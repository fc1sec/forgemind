## Key Features Delivered

### 1. Automatic Version Checking

**What it does:**
- Checks for new versions once per day (non-blocking)
- Shows update notifications with clear messaging
- Distinguishes safe (MINOR) from breaking (MAJOR) updates
- Stores check timestamp locally in `~/.forgemind/version_checks/`

**Files:**
- `forgemind/cli/version.py` — 150 LOC
- Integration in `forgemind/cli/main.py` — App callback

**Non-Blocking Design:**
- Fails silently on network errors
- Never interrupts user work
- Can be skipped with `--skip-version-check` flag

### 2. Backward Compatibility Guarantees

**What it does:**
- Documents which versions work together
- Explicitly lists compatible projects and outputs
- Provides rollback paths for each version
- All v1.x versions backward compatible within major version line

**Files:**
- `forgemind/version.py` — COMPATIBILITY dict

**Coverage:**
- v1.2.1 ↔ v1.2.0 ↔ v1.1.0: Full compatibility

## Technical Specifications

### Version Detection

```python
# forgemind/cli/version.py
get_installed_version()  # Reads from pip show output
get_latest_version()     # Queries PyPI JSON API
should_update(current, latest)  # Version comparison
is_breaking_change(from_version, to_version)  # Major version detection
should_check_version()   # Daily check interval logic
check_version_availability(skip=False)  # Main orchestration
```

### Compatibility Matrix

```python
# forgemind/version.py
COMPATIBILITY = {
    "1.2.1": {
        "works_with_projects": ["1.2.1", "1.2.0", "1.1.0"],
        "works_with_outputs": ["1.2.1", "1.2.0", "1.1.0"],
        "can_rollback_to": ["1.2.0", "1.1.0"],
        "backward_compatible": True,
    },
    # ... similar entries for v1.2.0, v1.1.0
}
```

### Notification Types

**MINOR Update (Safe):**
```
✓ Update available: v1.2.0 → v1.2.1

Backward compatible. Safe to update anytime.
Update: pip install --upgrade forgemind
```

**MAJOR Update (Breaking):**
```
⚠️  Major Update Available: v1.2.1 → v2.0.0

This is a MAJOR version update with breaking changes.

Before updating:
  1. Backup your .forgemind/ directory
  2. Read BREAKING_CHANGES.md
  3. Test in non-critical project first
  4. Update: pip install --upgrade forgemind
```

## Known Limitations

1. **PyPI Query:** Version checking requires internet connectivity
   - Gracefully fails if PyPI unavailable
   - Check happens once per day (cached locally)

2. **Version Inference:** Can only compare versions using packaging library
   - Supports semantic versioning (SemVer 2.0.0)
   - Invalid versions handled gracefully

3. **Network Timeout:** PyPI query has 5-second timeout
   - Prevents hanging if network is slow
   - Fails silently (user work continues)
